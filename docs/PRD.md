# PRD: 讀書筆記本系統

## 1. 專案概述

### 背景與動機
現代人閱讀書籍後常常忘記重點或缺乏歸納與分享的平台。為了解決這個問題，本專案將開發一個「讀書筆記本系統」，讓使用者可以方便地統一記錄所閱讀的書籍、撰寫心得、給予評分，並提供想參考書籍評價的閱讀愛好者一個查詢與交流的空間。

### 目標用戶
- 想要整理、發表自己讀書心得以及記錄閱讀歷程的人。
- 閱讀新書前，想要尋找並參考他人書評與心得的人。

### 核心價值主張
提供一個簡潔易用的平台，幫助使用者系統化地記錄閱讀歷程，並透過評分、分享以及留言互動機制，打造喜歡閱讀者的微型交流社群。

---

## 2. 功能需求

### 主要功能清單
1. 紀錄書名（新增書籍資訊）
2. 心得撰寫區（分享閱讀心得）
3. 為書評分（給予書籍評價等級）
4. 搜尋書籍（依據關鍵字尋找特定書籍）
5. 留言功能（與其他使用者的心得進行互動交流）

### 使用者故事 (User Stories)
- **作為閱讀愛好者**，我希望**能新增一本書的紀錄（包含書名）**，以便**集中管理我的閱讀書單**。
- **作為閱讀愛好者**，我希望**能有一個專屬撰寫區寫下心得**，以便**日後回顧這本書的精華與重點**。
- **作為評論者**，我希望**能為看過的書籍給予評分**，以便**直觀地表達我對這本書的評價與喜好程度**。
- **作為尋書人**，我希望**能在系統中輸入關鍵字搜尋書名**，以便**快速找到我想參考的特定書籍心得與評分**。
- **作為社群參與者**，我希望**能在別人的心得下方留下評論**，以便**與其他讀者分享觀點並討論該書籍**。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask
- **模板引擎**：Jinja2
- **資料庫**：SQLite

### 效能與安全考量
- **效能**：
  - 首頁載入時間與搜尋回應速度需保持在合理範圍，確保良好的使用者體驗。
  - SQLite 在初期輕量級訪問下應具備足夠效能。
- **安全性**：
  - 防範 SQL Injection：確保新增書籍、留言和搜尋輸入時，對資料庫的查詢與寫入皆使用參數化查詢等安全寫法。
  - 防範 XSS（跨站腳本攻擊）：針對使用者輸入的心得與留言內容，前端呈顯時必須進行安全過濾與跳脫 (Escape) 處理。

## 4. MVP 範圍

### Must Have (必須擁有)
- 新增包含書名與內容的讀書心得。
- 心得撰寫時能給予該書籍評分。
- 首頁能列出所有已經建檔的書籍／心得列表。

### Should Have (應該擁有)
- 書籍搜尋功能（依書名搜尋）。
- 在各別書籍的心得下方可新增留言。

### Nice to Have (未來可考慮擁有)
- 使用者註冊與登入（會員系統與個人頁面）。
- 依評分高低或時間對清單排序。
- 書籍分類或標記標籤（Tags）。
